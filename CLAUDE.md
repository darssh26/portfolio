# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev      # http://localhost:4321
npm run build    # → dist/
npm run preview  # serve the production build locally
```

There is no test suite, no linter, and no formatter configured. The only correctness check is `npm run build` (Astro + TypeScript strict). Run it before declaring work done.

## Architecture

Static, single-page Astro 5 site deployed to GitHub Pages. The whole site is one route (`src/pages/index.astro`) that composes section components in order: Hero → Summary → Experience → Projects → Skills → Education → Contact → Footer.

**Content is data, not markup.** Every section component reads from a typed module in `src/data/` (`site.ts`, `experience.ts`, `projects.ts`, `skills.ts`, `education.ts`). To change copy, edit the data file — do not edit the component. Components only exist to render the data shape; if the data shape changes, the component changes with it.

**Styling.** Tailwind 4 via the Vite plugin (`@tailwindcss/vite`), no `tailwind.config.js`. Global styles and theme tokens live in `src/styles/global.css`. Dark mode is class-based (`.dark` on `<html>`) and is set pre-paint by an inline script in `BaseLayout.astro` to avoid FOUC; `ThemeToggle.astro` flips it and persists to `localStorage`.

**SEO/metadata.** `BaseLayout.astro` owns the `<head>`: title, canonical, OpenGraph, Twitter card, and a `Person` JSON-LD schema sourced from `site.ts`. The sitemap is generated by `@astrojs/sitemap` and depends on `site` in `astro.config.mjs` being the deployed URL.

**Deployment.** Push to `main` triggers `.github/workflows/deploy.yml`, which runs `npm ci && npm run build` and publishes `dist/` to GitHub Pages. There is no preview/staging environment — `main` is production.

## Conventions

- New section on the page = new component in `src/components/` + new data module in `src/data/` + import into `src/pages/index.astro`. Keep sections self-contained.
- `public/cv.pdf` is the live CV link surfaced in the UI; replace the file in place when the CV changes (no rename).
- Project links in `src/data/projects.ts` use `"#"` as a placeholder for unfinished URLs — preserve that convention rather than removing the link entry.
